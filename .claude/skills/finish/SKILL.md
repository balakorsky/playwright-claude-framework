# Skill: finish

Завершение рабочей сессии. Вызывается командой `/finish`.

## Шаги выполнения

1. **Запусти smoke-тесты** — быстрая проверка что ничего не сломано:
   ```bash
   npx playwright test --grep "@smoke" --project=api
   npx playwright test --grep "@smoke" --project=chromium
   ```
   Если тесты падают — сообщи и **не продолжай** до исправления.

2. **Проверь git state** — `git status`
   - Если есть незакоммиченные изменения — спроси пользователя нужно ли коммитить
   - Если пользователь сказал "да" — сделай коммит с осмысленным сообщением по правилам из `.claude/rules/git.md`

3. **Проверь visual снапшоты** — `git status tests/visual/`
   - Если появились новые или изменённые PNG — напомни что нужно запустить "Update Visual Snapshots" workflow на GitHub для Linux-baseline

4. **Push** — если есть новые коммиты, спроси пользователя нужно ли пушить

5. **Итоговый отчёт**:

```
=== Сессия завершена ===
Smoke-тесты: ✓ прошли / ✗ N упало
Коммитов: N новых
Запушено: да / нет
Visual snapshots: актуальны / требуют Linux update

До следующей сессии!
```

## Правила

- Smoke-тесты обязательны — не пропускай шаг 1
- Не делай `--update-snapshots` автоматически
- Не пуши без явного согласия пользователя
- Если smoke-тесты упали — останови сессию, не коммить сломанный код
