# Skill: finish

Завершение рабочей сессии. Вызывается командой `/finish`.

## Шаги выполнения

1. **Запусти smoke-тесты** — быстрая проверка что ничего не сломано:
   ```bash
   npx playwright test --grep "@smoke" --project=api
   npx playwright test --grep "@smoke" --project=chromium
   ```
   Если тесты падают — сообщи и **не продолжай** до исправления.

2. **Проверь git state** — `git status`
   - Если есть незакоммиченные изменения — спроси пользователя нужно ли коммитить
   - Если пользователь сказал "да" — сделай коммит по правилам из `.claude/rules/git.md`

3. **Проверь visual снапшоты** — `git status tests/visual/`
   - Если появились новые или изменённые PNG — напомни про "Update Visual Snapshots" workflow

4. **Обнови SNAPSHOT.md** — запиши текущее состояние проекта:
   - Прочитай `.claude/SNAPSHOT.md`
   - Обнови секцию `## Последняя сессия` — дата, что сделано, ветка, статус тестов
   - Обнови `## Приоритеты` если пользователь назвал следующие задачи
   - Обнови `## Известные проблемы` если появились новые или закрылись старые
   - Закоммить изменения в SNAPSHOT.md отдельно: `Update SNAPSHOT.md`

5. **Push** — если есть новые коммиты, спроси пользователя нужно ли пушить

6. **Итоговый отчёт**:

```
=== Сессия завершена ===
Smoke-тесты: ✓ прошли / ✗ N упало
Коммитов: N новых
Запушено: да / нет
Visual snapshots: актуальны / требуют Linux update
SNAPSHOT.md: обновлён

До следующей сессии!
```

## Правила

- Smoke-тесты обязательны — не пропускай шаг 1
- Не делай `--update-snapshots` автоматически
- Не пуши без явного согласия пользователя
- Если smoke-тесты упали — останови сессию, не коммить сломанный код
- SNAPSHOT.md всегда обновляется последним, после всех коммитов
